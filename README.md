# Auto Research Idea

Turn a rough **meta-idea** into a ranked list of 50–100 concrete, novel **paper
ideas** — generated by a pipeline of Claude Code **subagents**, one per step, with
a **live web dashboard** that shows each step's work as it happens.

You drive it by talking to **Claude Code** (the orchestrator); a read-only web
page visualizes progress.

```
brainstorm ─▶ retrieve ─▶ digest ─▶ hybridize (×N parallel) ─▶ prioritize
 (opus)       (sonnet)    (sonnet)   (opus)                     (opus)
   │            │           │          │                          │
 10 variants  ~50 papers  genes     50–100 ideas              scored & ranked
 + queries    (dedup,     (parallel  + key insights
              ranked)     analysis)
```

## Architecture

- **Orchestrator** — Claude Code, running the `research-ideas` skill
  (`.claude/skills/research-ideas/`). It delegates each step to a subagent and
  records progress in `runs/<id>/status.json`.
- **One subagent per step** (`.claude/agents/*.md`) — each independently
  optimizable, each writes its artifact to the run dir:
  - `idea-brainstormer` (opus) → `brainstorm.json` — 10 variants + search queries
  - `paper-retriever` (sonnet) → `papers.json` — runs the search tool (~50 papers)
  - `paper-digester` (sonnet) → `genes.json` — runs the digest tool (parallel)
  - `idea-hybridizer` (opus, ×N parallel) → `ideas_raw_*.json` — 50–100 ideas
  - `idea-prioritizer` (opus) → `ideas.json` — scored & ranked
- **Python tools** (`auto_research_idea/`) — the mechanical, parallel work:
  `search` (multi-source retrieval + dedup + rank) and `digest` (parallel paper
  analysis, on Sonnet to save cost).
- **Dashboard** (`auto_research_idea/dashboard.py`) — read-only web page that
  polls the run artifacts and renders variants, papers, key insights, and ranked
  ideas live. It never runs the pipeline.

**Models:** the cheap mechanical steps (retrieve, digest) use **Sonnet**; the
creative/judgment steps (brainstorm, hybridize, prioritize) use **Opus**. Change a
step's model in its `.claude/agents/*.md` file (or `config.yaml` for digest).

## Setup

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
cp .env.example .env          # put your real ANTHROPIC_API_KEY (and optional S2 key) here
```

`ANTHROPIC_API_KEY` is required (the digest tool + subagents use it). A
`SEMANTIC_SCHOLAR_API_KEY` and `CONTACT_EMAIL` are optional and only raise rate
limits. **Secrets go in `.env`, not `.env.example`** — `python-dotenv` loads
`.env`, and `.env` is git-ignored.

## Usage

1. **Start the dashboard** (separate terminal):
   ```bash
   .venv/bin/python -m auto_research_idea.dashboard
   ```
   Open <http://localhost:8000>.

2. **Run the pipeline** in Claude Code (in this directory). Either:
   - tell it your idea directly:
     > use the research-ideas skill on: diffusion models for combinatorial optimization
   - or type the idea into the dashboard's box to **queue** it, then tell Claude
     Code: *"run the queued research-ideas request"* (it finds it via
     `runstate pending`).

   Claude Code brainstorms variants, spawns the retrieve/digest subagents, fans
   out parallel hybridizers, prioritizes, and updates the dashboard live. The
   final ranked ideas land in `runs/<id>/ideas.json` and on the page.

### Running the Python tools directly (debugging)

```bash
.venv/bin/python -m auto_research_idea.search --queries "graph neural network CO" --out papers.json
.venv/bin/python -m auto_research_idea.digest --papers papers.json --out genes.json
```

## Configuration

`config.yaml` tunes the **tools**: paper sources, retrieval limits
(`retrieval.max_papers` ≈ 50), and the digest model/effort. Orchestration knobs
(10 variants, number of parallel hybridizers → 50–100 ideas) live in the skill and
agent files.

## Project layout

```
.claude/
  skills/research-ideas/SKILL.md     # orchestrator
  agents/                            # one subagent per step
    idea-brainstormer.md  paper-retriever.md  paper-digester.md
    idea-hybridizer.md    idea-prioritizer.md
auto_research_idea/
  search.py        # tool: queries -> papers.json (search + dedup + rank)
  digest.py        # tool: papers.json -> genes.json (parallel analysis, Sonnet)
  dashboard.py     # read-only live web dashboard (stdlib only)
  runstate.py      # run-dir status contract (orchestrator <-> dashboard)
  llm.py models.py config.py
  sources/         # arxiv, semantic_scholar, openalex, _http (backoff), registry
config.yaml  requirements.txt
runs/<id>/         # per-run artifacts (created at runtime; git-ignored)
```

Add a paper source by subclassing `PaperSource` in `sources/`. Optimize a step by
editing its `.claude/agents/*.md`. Change the run-dir contract in `runstate.py`
(the dashboard reads the same shapes).
